## Database Setup (Supabase)

This project is prepared to move LMS progress from local storage to a real database.

1. Copy environment template:

```bash
cp .env.example .env.local
```

1. Fill these values in `.env.local`:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

1. In Supabase Auth settings, enable anonymous sign-ins if you want progress
   to be persisted before explicit user registration/login.

1. Open your Supabase SQL editor and run:

- `db/schema.sql`

1. Use helper clients:

- Browser client: `lib/supabase/client.ts`
- Server client (SSR): `lib/supabase/server.ts`
- Admin client (service role): `lib/supabase/admin.ts`
